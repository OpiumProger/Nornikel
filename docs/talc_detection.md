# Определение доли талька на OM-снимках

Документ описывает актуальный пайплайн количественной оценки талька в проекте **Норникель Hackathon** (июль 2026).

## Кратко

**Доля талька** — площадь маски талька относительно всего кадра:

```
talc_pct = (число пикселей маски талька / число пикселей изображения) × 100%
```

Порог из ТЗ: **`talc_pct > 10%` → оталькованная руда**.

Финальный класс руды определяется **экспертными правилами**, не CNN (CNN — только для сравнения).

---

## Архитектура (гибрид)

```
OM-снимок
    │
    ├─► U-Net (EfficientNet-B0) ──► prob_map ──► порог ──► unet_mask
    │
    └─► dark_gray_phase (эвристика) ─────────────► dark_mask
                              │
                              ▼
                    talc_mask = unet_mask OR dark_mask
                              │
                              ▼
                         talc_pct (%)
```

### Почему гибрид, а не только U-Net

U-Net обучена на **псевдо-масках из синей экспертной разметки** (332 пары). На панорамах без синих контуров модель часто **недооценивает** тёмно-серый тальк.

Поэтому в продакшен-пайплайне синяя разметка **не используется**. Вместо неё — детектор **`dark_gray_phase`**: ищет тёмно-серую матовую фазу в матрице руды.

Итоговая маска:

```python
talc_mask = np.maximum(unet_mask, dark_mask)   # логическое OR
```

Метод в отчёте: `unet+dark_gray_phase` или `unet_tiled_stitch+dark_gray_phase` (для панорам).

## Модуль 1: U-Net

| Параметр | Значение |
|----------|----------|
| Архитектура | `smp.Unet` + `efficientnet-b0` |
| Веса | `models/best_talc_unet.pth` (не в git, через Release/облако) |
| Конфиг | `models/talc_unet_config.json` |
| Порог (калиброванный) | **0.05** (val Dice ≈ 0.55 на 49 val-парах) |
| Патч инференса | 256×256, stride 128 |
| Постобработка | MORPH_CLOSE + фильтр по площади (min 120 px) |

### Порог

- **`auto`** (по умолчанию) — берёт калиброванный порог из конфига, на больших кадрах дополнительно подстраивает по перцентилям `prob_map`.
- Фиксированный: `--talc-threshold 0.08`

Калибровка:

```powershell
python scripts/calibrate_talc_unet.py
```

Перезаписывает `models/talc_unet_config.json`.

### Тайловый инференс

На больших снимках U-Net не даунскейлит весь кадр целиком, а режет на патчи 256×256 с перекрытием и усредняет `prob_map`.

## Модуль 2: `dark_gray_phase`

Файл: `src/talc_mask.py` → `extract_dark_talc_mask()`

Ищет **тёмно-серую / слабо окрашенную матовую фазу**:

| Включаем | Исключаем |
|----------|-----------|
| Средне-тёмная яркость (перцентили L 18–62%) | Яркие сульфиды (верхние 12% яркости) |
| Низкая насыщенность (HSV) | Самый чёрный фон (нижние 8%) |
| Низкая цветность (LAB chroma) | Синие/голубые линии разметки |
| Связные области ≥ min_area | Слишком крупные компоненты (>12% кадра) |

Защита от «заливки» всей матрицы: если маска >35% площади — применяется более строгий диапазон яркости (25–55 перцентиль).

**Важно:** это эвристика, не ground truth. На панорамах 4–8.jpg даёт **1.5–2.3%** талька (против ~0.5–1.6% только U-Net).

## Режимы для панорам

Скрипт: `scripts/analyze_panorama.py`

| Режим | Флаг | Скорость | Точность |
|-------|------|----------|----------|
| **downscale** | `--talc-mode downscale` | ~3 мин / фото | Быстрый даунскейл до `--max-side` (4096), затем гибрид |
| **tiled_stitch** | `--talc-mode tiled_stitch` | ~7–30 мин / фото | Нарезка 1536×1536, U-Net по тайлам, склейка до 8192 px + feathering |

Примеры:

```powershell
# Одна панорама, быстрый режим
python scripts/analyze_panorama.py "data/raw/Панорамы/11.jpg" -o reports/out --talc-mode downscale

# Точный тайловый режим
python scripts/analyze_panorama.py "data/raw/Панорамы/11.jpg" -o reports/out --talc-mode tiled_stitch

# Пакет + CNN для сравнения
python scripts/analyze_panorama.py --all --limit 5 -o reports/panoramas_batch5_v2 --talc-mode downscale --max-side 4096 --use-cnn
```

## Известные ограничения

1. **U-Net** — слабая генерализация на панорамы без синей разметки.
2. **`dark_gray_phase`** — эвристика; может пере/недооценивать тальк на тёмных или неоднородно освещённых кадрах.
3. **Сульфиды** — отдельная эвристика (`src/sulfide_mask.py`); сейчас почти всё попадает в «тонкие» (96–100%).
4. **CNN ResNet18** на панорамах часто даёт «оталькованная» при низком `talc_pct` — **не использовать как финальный класс**.
5. **`tiled_stitch`** точнее, но очень медленный на гигапиксельных панорамах.

### Возможные улучшения

- Переобучить U-Net на масках `dark_gray_phase` вместо синей разметки.
- Подкрутить пороги сульфидов (крупные зёрна → обычные срастания).
- Ручная валидация 2–3 панорам с экспертом для калибровки эвристик.
